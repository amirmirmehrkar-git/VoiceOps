# VoiceOps Demo Repository Checklist

## Pre-Demo Setup

### ✅ Environment
- [ ] VAPI account configured
- [ ] API keys set in environment
- [ ] Test Jira project created
- [ ] Test PagerDuty service configured
- [ ] Webhook URLs configured

### ✅ Data
- [ ] Demo transcripts ready
- [ ] Sample incidents prepared
- [ ] Test voice recordings available
- [ ] Backup JSON files ready

### ✅ Integration
- [ ] Jira webhook tested
- [ ] PagerDuty webhook tested
- [ ] Schema validation working
- [ ] Error handling tested

## Demo Flow Checklist

### Opening (5s)
- [ ] Logo/branding visible
- [ ] Tagline displayed: "Voice is the input. Production-grade incidents are the output."

### Step 1: Voice Input (15s)
- [ ] Voice interface ready
- [ ] Test call/voice input available
- [ ] Real-time transcription visible

### Step 2: AI Processing (20s)
- [ ] VAPI agent active
- [ ] JSON generation visible
- [ ] Schema validation passing

### Step 3: Output Display (20s)
- [ ] Final JSON displayed
- [ ] Severity highlighted
- [ ] All fields populated

### Step 4: Integration (20s)
- [ ] Jira ticket created
- [ ] PagerDuty alert triggered
- [ ] Confirmation displayed

### Closing (10s)
- [ ] Key benefits summarized
- [ ] Next steps mentioned

## Backup Plan

### If Live Demo Fails
- [ ] Pre-recorded video ready
- [ ] Offline JSON files available
- [ ] Screenshots prepared
- [ ] Transcript files ready

## Post-Demo

- [ ] Q&A answers prepared
- [ ] Contact information ready
- [ ] Follow-up materials prepared
