# VoiceOps Compliance Disclaimers

## Healthcare Disclaimer

**Important**: VoiceOps is a technology platform for incident management. When used in healthcare contexts:

- VoiceOps does NOT process Protected Health Information (PHI) by default
- VoiceOps does NOT provide medical advice or clinical decision support
- VoiceOps does NOT replace HIPAA-compliant medical record systems

**Customer Responsibilities**:
- Ensure appropriate use of VoiceOps in healthcare contexts
- Obtain necessary authorizations and approvals
- Maintain HIPAA compliance independently
- Use VoiceOps only for IT/operational incidents, not patient data

**VoiceOps Capabilities**:
- Can be configured for HIPAA-aligned use cases
- Business Associate Agreement (BAA) available for enterprise customers
- Encryption and security controls in place
- Audit logging for compliance

**Recommendation**: Consult with legal/compliance teams before using VoiceOps in healthcare environments.

---

## Security Disclaimer

**Important**: VoiceOps is designed for incident management, not as a primary security tool.

**Limitations**:
- VoiceOps does NOT replace Security Information and Event Management (SIEM) systems
- VoiceOps does NOT provide real-time threat detection
- VoiceOps does NOT perform security analysis or forensics

**VoiceOps Role**:
- Incident reporting and ticketing
- Structured incident data capture
- Integration with security tools (SIEM, SOAR)
- Audit trail for incident response

**Best Practices**:
- Use VoiceOps as part of a comprehensive security program
- Integrate with existing security tools
- Maintain separate security monitoring systems
- Follow NIST incident response framework
